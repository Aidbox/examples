Measure/$evaluate-measure via SQL

FHIR Measure/$evaluate-measure implemented as native PostgreSQL SQL on Aidbox. No CQL engine required. 12 CMS quality measures, 484 test patients, interactive demo.

01 Quick Start

Prerequisites: Docker with Docker Compose, Python 3, curl (optional, for manual API calls).

Build and start the full stack:

bash 
docker compose up --build

This starts three containers: PostgreSQL, Aidbox (port 8888), and the SQL evaluate app (port 8090).

02 Activate Aidbox

Open http://localhost:8888 in your browser and follow the activation prompt.

One-time step. Aidbox requires browser activation on first launch. After activation, data persists across restarts.

03 Load Data

In a second terminal, from this directory:

bash 
python3 setup.py

This creates shared views, loads terminology and clinical data for all 12 measures (~2 min).

Idempotent. Safe to run multiple times — views are replaced, data is upserted via PUT.

04 Evaluate a Measure

Evaluate CMS130 for a single patient:

bash 
curl -u root:secret -X POST \
  'http://localhost:8888/Measure/$evaluate-measure?measure=cms130&subject=Patient/007ec5f1-08cf-474a-a472-f6a92cca4b79&periodStart=2026-01-01&periodEnd=2026-12-31'

05 REST Console Examples

Open the Aidbox REST Console at http://localhost:8888/ui/console and paste any of the snippets below. The adapter resolves the measure parameter in three forms: short alias (cms130), FHIR Resource ID (CMS130FHIRColorectalCancerScrn), or canonical URL.

By alias — individual patient

CMS130 Colorectal Cancer Screening for a single patient:

rest console 
POST /Measure/$evaluate-measure?measure=cms130&subject=Patient/007ec5f1-08cf-474a-a472-f6a92cca4b79

By canonical URL — summary (all patients)

CMS125 Breast Cancer Screening, population-level MeasureReport:

rest console 
POST /Measure/$evaluate-measure?measure=https://madie.cms.gov/Measure/CMS125FHIRBreastCancerScreen

By alias with explicit period

CMS165 Controlling High Blood Pressure for a single patient, measurement period 2026:

rest console 
POST /Measure/$evaluate-measure?measure=cms165&subject=Patient/045f7e0b-bfb3-4ee0-a06d-83c853f6a81e&periodStart=2026-01-01&periodEnd=2026-12-31

By FHIR Resource ID — individual patient

CMS131 Diabetes Eye Exam for a single patient:

rest console 
POST /Measure/$evaluate-measure?measure=CMS131FHIRDiabetesEyeExam&subject=Patient/01a1241d-fd97-4c72-b288-fd31c4c7ae80
Tip. Omit subject for a summary report across all patients. Omit periodStart/periodEnd to use defaults (2026-01-01 to 2026-12-31).

06 Demo App

Start a local HTTP server from this directory:

bash 
python3 -m http.server 3000

Then open the demo in your browser:

→ http://localhost:3000/demo/app.html

The demo shows all 12 measures with:

  • Patient-level care gaps
  • Step-by-step decision chains with evidence
  • Measure scores (population-level)
Note. The demo app uses the Aidbox $sql endpoint directly for fast batch evaluation across all patients. For production integration, use the Measure/$evaluate-measure FHIR API described above.

⚙ Architecture

Aidbox JSONB → 8 flat views → concepts table → measure SQL CTEs → MeasureReport
POST /Measure/$evaluate-measure  →  Flask app reads SQL  →  executes on PostgreSQL  →  builds FHIR MeasureReport

Each measure is a single SQL file (~200–400 lines) that uses shared flat views and a concepts table for offline terminology matching. No CQL engine, no Java, no external terminology service calls at runtime.

How it works

  1. Client sends POST /Measure/$evaluate-measure?measure=cms130&subject=Patient/123
  2. Aidbox routes the custom operation to the Flask app (port 8090)
  3. Flask app reads the measure SQL file, substitutes parameters, executes via Aidbox $sql
  4. SQL query runs over 8 shared flat views (patient, encounter, condition, observation, etc.) and the concepts table
  5. Results are assembled into a FHIR MeasureReport and returned to the client

⚙ API Parameters

■ Shutdown

Stop all containers (data preserved in Docker volume):

bash 
docker compose down

Or stop and delete all data (clean slate for next run):

bash 
docker compose down -v
